##  Why This Exists

I'm a Flutter developer who has sat through dozens of interviews — on both sides of the table. The same questions kept coming up, so I compiled them into **short 2–3 line answers** built for last-minute revision.

This isn't a textbook. It's a **cheat sheet built from real interview experience** — now grown to **256 questions** across two PDFs.

##  How to Prepare

A PDF alone won't get you the job — **how you use it matters**.

**Week 1–2 · Build the Foundation**
> Read 15–20 questions daily. Don't memorize — understand the *why* behind each answer. If something feels unclear, open your IDE and write a small example.

**Week 3 · Active Recall**
> Cover the answers and explain each concept out loud. If you stumble, that's the topic you need to revisit.

**Ongoing · Grow Your Own Bank**
> Every time you face a new question in an interview, add it here via a PR. The best question banks are the ones the community builds together.

###  Pro Tips

- **Pair with code.** For any question about widgets or state management, build a 20-line example — reading about `InheritedWidget` is not the same as using one.
- **Explain to someone else.** If you can teach it simply, you understand it.
- **Know your level.** PDF 1 → mid-level interviews. PDF 2 → senior/lead/architect interviews.
- **Track weak spots.** Highlight questions you stumble on and revisit them the day before each interview.

##  Contributing

This project gets better with every contribution.

1. **Fork** this repository
2. **Create a branch** — `git checkout -b add-new-questions`
3. **Add your questions** — follow the existing format (question + 2–3 line answer)
4. **Submit a Pull Request** with a brief description of what you added

**Ideas for contributions:**
- Questions from interviews you recently attended
- Corrections or improvements to existing answers
- New topic categories (e.g., Flutter AI/ML, Flame game engine)
- Translations into other languages

> Please keep answers concise — 2–3 lines max. The goal is fast revision, not deep dives.
