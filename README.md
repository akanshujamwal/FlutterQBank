## 👋 Why This Exists

I'm a Flutter developer who has sat through dozens of interviews — on both sides of the table. I kept seeing the same questions come up again and again, so I compiled them into a single PDF with **short 2–3 line answers** that are perfect for last-minute revision.

This isn't a textbook. It's a **cheat sheet built from real interview experience**.

## 🧠 How to Prepare

A PDF alone won't get you the job — **how you use it matters**. Here's a method that works:

**Week 1–2: Build the Foundation**
> Read 15–20 questions daily. Don't memorize — understand the *why* behind each answer. If something feels unclear, open your IDE and write a small example.

**Week 3: Active Recall**
> Cover the answers and try to explain each concept out loud. If you stumble, that's the topic you need to revisit.

**Ongoing: Grow Your Own Bank**
> Every time you face a new question in an interview, add it to your personal copy. The best question banks are the ones you build yourself.

### 💡 Pro Tips

- **Pair with code.** For any question about widgets or state management, build a 20-line example. Reading about `InheritedWidget` is not the same as using one.
- **Explain to someone else.** If you can teach it simply, you understand it.
- **Track weak spots.** Highlight questions you get wrong and revisit them before every interview.
